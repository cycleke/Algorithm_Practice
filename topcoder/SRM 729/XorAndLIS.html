SRM 729 1 800 XorAndLIS

Problem Statement

Let LIS(S) denote the length of the longest increasing subsequence of the sequence S. Note that the subsequence does not have to be contiguous. For example, for S1 = {10,0,20,0,30} we have LIS(S1) = 3 and for S2 = {1,1,1} we have LIS(S2) = 1.

You are given a long[] x: a sequence of nonnegative integers. You are allowed to modify this sequence by performing a sequence of zero or more updates. In each update, you select an index i between 0 and |x|-2, inclusive, and you change x[i+1] into (x[i+1] xor x[i]).

Your goal is to make LIS(x) as large as possible. Please compute and return this maximal value.

Definition

  • Class XorAndLIS
  • Method maximalLength
  • Parameters vector<long long>
  • Returns int
  • Method signature int maximalLength(vector<long long> x)
(be sure your method is public)

Limits

  • Time limit (s) 2.000
  • Memory limit (MB) 256

Constraints

  • x will contain between 1 and 100 elements, inclusive.
  • Each element in x will be between 0 and 10^18, inclusive.

Test cases

  1. input
    • x { 4LL, 2LL, 1LL }
    output
    Returns 3
    note
    You can do x[1] = x[1] xor x[0], the sequence will be {4,6,1}. Then you do x[2] = x[2] xor x[1] and the sequence becomes {4,6,7}.
  2. input
    • x { 0LL, 0LL, 0LL, 0LL, 0LL, 0LL, 0LL, 0LL, 0LL, 0LL }
    output
    Returns 1
    note
    Whatever you do, the sequence never changes, and the LIS is 1.
  3. input
    • x { 1LL, 2LL, 3LL, 4LL, 5LL, 6LL, 7LL, 8LL, 9LL, 10LL }
    output
    Returns 10
    note
    This time you don't need to do anything.
  4. input
    • x { 7LL, 3LL, 1LL }
    output
    Returns 2
  5. input
    • x { 1008LL, 42LL, 7LL, 5LL, 2LL, 9LL, 75LL, 0LL, 0LL, 12LL, 3LL, 6LL, 81LL, 4LL, 3LL }
    output
    Returns 12
  6. input
    • x { 1000000000000000000LL }
    output
    Returns 1
  7. input
    • x { 10000000000LL, 10000000001LL }
    output
    Returns 2
